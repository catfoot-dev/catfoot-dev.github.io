## 현재 상태

MIDI Editor는 단순한 실험용 스케치를 넘어, **MIDI 파일을 열고 재생 상태를 만들고 트랙과 노트를 함께 보는 앱**의 형태까지는 분명하게 들어왔다. 이날 들어간 작업은 눈에 띄는 대형 기능 추가보다는, 지금까지 쌓은 구조를 읽기 쉽고 다루기 쉽게 정리하는 쪽에 가까웠다.

같은 날 오전에는 import 정리와 코드 포맷 개선이 여러 파일에 걸쳐 들어갔고, 키보드 쪽의 옥타브 계산도 함께 다듬어졌다. 이어서 노트 그리드에서는 노트 색상을 생성 함수로 통일해, 트랙과 channel 정보를 기준으로 더 일관된 색을 쓰도록 정리했다. 이런 변화는 작은 수정처럼 보이지만, 화면 해석과 유지보수에는 꽤 직접적인 영향을 준다.

## 지금까지 만들어 둔 것

현재 커밋 기준으로 확인되는 기반은 꽤 분명하다.

- `menu`, `transport`, `track_list`, `keyboard`, `note_grid`, `attributes` 로 이어지는 기본 편집 화면
- MIDI 파일을 열고 로딩 결과를 반영하는 흐름
- 오디오 재생 상태를 공유하고 화면의 재생 커서와 연결하는 구조
- 트랙 선택과 solo/mute 같은 상태를 UI와 재생 쪽에 함께 반영하는 방식
- 노트를 화면에 올릴 때 트랙 단위로 해석하기 위한 데이터 흐름

이 프로젝트는 여전히 `WIP / Experimental` 상태다. 다만 이제는 “무엇을 만들지 정하지 못한 상태”는 아니고, 어떤 축으로 계속 확장할지가 비교적 명확해진 상태에 더 가깝다.

## 아직 실험 단계라고 보는 이유

지금 구조만으로도 프로젝트의 방향은 보인다. 하지만 아직 `v1.0` 이전의 실험 단계라는 표현이 어울리는 이유도 분명하다.

첫째, 현재 구조는 이미 편집기를 지향하고 있지만, 사용 경험 전체가 안정됐다고 보기는 어렵다.
둘째, README에 적어 둔 항목 중에서도 `quantize / humanize`, `tempo & time signature` 처리는 아직 `planned / partial` 상태로 남아 있다.
셋째, 지금은 기능을 늘리는 것보다 데이터를 어떻게 안정적으로 다룰지와 화면을 어떻게 일관되게 보여 줄지를 정리하는 과정에 더 가깝다.

## 앞으로 작업할 수 있는 것

현재 구조를 기준으로 다음 단계에서 현실적으로 이어 갈 수 있는 작업은 아래와 같다.

- 피아노 롤 편집 경험 개선
- 이벤트 단위의 정밀 편집 보강
- quantize / humanize 유틸리티 보강
- tempo / time signature 처리 확장
- import / export 안정화

이 다섯 가지는 서로 따로 놀지 않는다. 트랙 중심 구조와 현재의 UI 패널 구성이 이미 준비되어 있기 때문에, 이제는 “무엇을 더 만들 것인가”보다 “어떤 순서로 더 안정적으로 붙일 것인가”가 더 중요한 시점에 들어온 셈이다.

## 마무리

MIDI Editor는 아직 완성된 도구가 아니다. 앞으로 편집기답게 확장해 나갈 수 있는 최소한의 기반과 방향을 확보한 상태라고 볼 수 있다. 당분간은 기능을 무턱대고 늘리기보다, 이미 만든 구조를 흔들리지 않게 다지고 그 위에 편집 경험을 차근차근 쌓아 가는 쪽으로 진행하게 될 것 같다.
